This an attempt to learn and practice git in a simple way.


Step 1: Create a local repository.

I created a simple html file (this). Added some content to it.

Then I run this command from vscode terminal:
# git init


Step 2: Create a remote repository in Github and Push changes.

I then created a remote Github repo and added the file(s) to the remote github repo. Over here, the local repository will keep track of the changes in the remote branch (main).

# git add .
# git commit -m "first commit"
# git branch -M main
# git remote add origin https://github.com/afrin321/Practicing-Git.git
# git push -u origin main


Step 3: Pull and Push changes.

Next, I added a readme.md file from the remote github repository. This created a change in the remote and local repository. Now I first pull the change from the remote repo. By this time, I have also made some changes, so now I will push these changes into remote repo.

# git pull origin main
# git add .
# git commit -m "updated html file"
# git push origin main


Step 4: Create a new branch

Now, I want to test some styling with html page. For this I create a new branch. First, I created a new brach called dev-styles and then moved to that branch with the checkout command. Then, I added the styles and pushed changes to the new branch.

# git branch dev-styles
# git checkout dev-styles
# git add .
# git commit -m "added styles"
# git push origin dev-styles


Step 5: Merge changes

Once the new branch dev-styles is created and I am happy with the changes, I want to merge the changes of both branches into the 'main' branch. Here we will see how to merge changes and in the next step we will see how to deal with "merge conflicts".

To merge changes, we need to need to be in the branch we want to merge changes into. However, we must push any changes from the current branch before checkout so that the changes are not lost.

# git add .
# git commit -m "updated html - step 5"
# git push origin dev-styles

# git checkout main
# git push origin main


Step 6: Resolve conflicts

To create a conflict, I will create add a style to both branches but the same line will be different in each. After push changes in both branches, I will try to merge branch dev-styles into main.

I will set the class .mb 's margin-bottom as 100px in main and 50px in dev-styles. After getting the conflict, I manually changed the value to 75px in main branch. Committed the changes, tested and pushed to main branch.

# git add .
# git commit -m "added .mb - 100px"
# git push origin main

# git checkout dev-styles
# git add .
# git commit -m "added .mb - 50px"
# git push origin dev-styles

# git checkout main
# git add .
# git commit -m "Resolve merge conflict"
# git push origin main


Step 7: Revert changes

Next we will see how to go back to our previous stage aka undo/revert changes. Now, let's undo the changes from the previous merge and go back to margin-bottom of 100px

The commit-hash of the chosen version is e976b315e11b10de7efe124e34999b4536f86b91.

# git revert e976b315e11b10de7efe124e34999b4536f86b91

Faced a merge conflict when resolving and then manually merged the files and commited changes.
